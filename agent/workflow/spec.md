# Short Code: /spec
**Trigger:** When user inputs `/spec [description]`.

## Specification Protocol
1. **SDD Gatekeeper Check:** Ensure request follows SDD.
2. **Create Spec:** Create `docs/specs/SPEC-00X_Name.md`.
3. **Draft PRD:** Write constraints, user stories, and acceptance criteria.
4. **Review:** Ask user to review the spec before planning.
